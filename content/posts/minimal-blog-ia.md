# 문서형 블로그 IA를 먼저 설계한 이유

개인 기술 블로그는 처음에는 글이 몇 개 없어서 단순해 보이지만, 시간이 지나면 분류 체계가 느슨할수록 다시 찾기 어려워집니다. 그래서 홈 화면보다 먼저 **탐색 구조**를 잡는 편이 낫습니다.

## 왜 홈보다 구조가 먼저인가

- 블로그의 핵심 사용자는 미래의 나 자신입니다.
- 글이 많아질수록 태그와 카테고리의 일관성이 중요해집니다.
- 추천 글보다 최근 글, 관련 글, 분류 이동이 더 자주 사용됩니다.

## 1차 버전에서 중요한 기준

이번 구조는 과한 카드 UI를 피하고, 제목과 요약을 중심으로 빠르게 읽을 수 있게 설계합니다. 문서형 블로그에서는 리스트의 밀도와 본문 폭이 읽기 경험을 크게 좌우합니다.

```tsx
export function PostListItem() {
  return (
    <article className="border-b py-6">
      <h2>Readable title first</h2>
    </article>
  );
}
```
