# Next.js App Router에서 블로그 골격을 잡을 때의 기준

App Router는 레이아웃과 페이지 경계를 명확하게 나누기 좋기 때문에 아카이브형 블로그에 잘 맞습니다. 특히 공통 헤더, 메타데이터, 정적 생성을 한곳에서 다루기 쉽습니다.

## 추천하는 라우팅 기준

1. 홈은 최신 글과 대표 카테고리 중심으로 단순하게 유지합니다.
2. 전체 목록은 탐색 허브 역할을 하도록 만듭니다.
3. 상세 페이지는 목차와 관련 글을 통해 재탐색을 돕습니다.

## SEO를 어디서 다루는가

기본 메타데이터는 `layout.tsx`에서 관리하고, 글 단위 메타데이터는 `generateMetadata`에서 세부적으로 덮어쓰는 방식이 가장 단순합니다.

## 콘텐츠 저장 전략

초기에는 markdown 파일만으로도 충분합니다. frontmatter와 본문을 나누고, 렌더링 규칙을 내부 유틸로 관리하면 의존성을 늘리지 않고도 시작할 수 있습니다.
