# 온톨로지 · 토폴로지 · 택소노미

`-logy`로 끝나는 이 단어들은 발음이 비슷해서 자주 섞여 쓰인다. 그런데 세 용어는 어원도, 다루는 분야도, "구조의 어떤 면을 보는가"도 전부 다르다. 한 문장으로 먼저 못 박으면 이렇다.

- **택소노미(Taxonomy)** — 위에서 아래로 줄 세우는 **계층 분류**
- **토폴로지(Topology)** — 누가 누구와 이어졌는지의 **연결 구조**
- **온톨로지(Ontology)** — 무엇이 존재하고 어떤 관계인지의 **의미 망**

아래에서 하나씩, 비유와 함께 풀어 본다.

---

## 택소노미 — 서랍에 줄 세우기

택소노미(taxis: 배열 + nomos: 규칙)는 가장 익숙한 구조다. **하나의 부모 아래 자식이 갈라지는 트리(tree)** 형태다.

생물 분류가 교과서적인 예다. `동물 > 척추동물 > 포유류 > 개`처럼, 위로 갈수록 넓고 아래로 갈수록 좁아진다. 도서관 청구기호, 파일 폴더, 블로그 카테고리도 전부 택소노미다.

```
동물
├─ 포유류
│  ├─ 개
│  └─ 고양이
└─ 조류
   └─ 참새
```

핵심 특징은 **"한 갈래는 한 부모만 가진다"**는 점이다. 개는 포유류 아래에만 있다. 그래서 단순하고 직관적이지만, "반려동물"처럼 여러 갈래에 걸치는 개념은 어디에 넣을지 애매해진다.

> 한 줄 요약: **위아래 줄 세우기.** 분류는 깔끔하지만 관계는 못 담는다.

---

## 토폴로지 — 누가 누구와 이어졌나

토폴로지(topos: 장소 + logos: 학문)는 두 갈래로 이해하면 된다.

### 1) 수학에서의 위상수학

"늘리고 구부려도 변하지 않는 성질"을 다루는 분야다. 유명한 비유가 **도넛과 머그컵**이다. 둘 다 "구멍이 하나"라는 점에서 위상적으로 같다. 길이나 각도는 무시하고, **연결과 연속성**만 본다.

### 2) IT에서의 네트워크 토폴로지

실무에서 더 자주 만나는 쪽이다. **노드(컴퓨터·서버·라우터)들이 어떻게 연결되어 있는가**의 배치도를 뜻한다. 성형(star), 링형(ring), 메시(mesh) 등이 있다.

```
[성형 star]          [메시 mesh]
    A                  A───B
    │                  │ ╳ │
B───H───C              C───D
    │
    D
```

택소노미가 "위아래"라면 토폴로지는 **방향이 없는 연결**이다. 부모-자식이 아니라, 옆으로 이어진 관계망을 본다.

> 한 줄 요약: **연결 지도.** 무엇이 무엇과 닿아 있는지를 본다.

---

## 온톨로지 — 의미까지 적어 두기

온톨로지(onto: 존재 + logos: 학문)는 원래 철학 용어였지만, 지금은 AI·시맨틱 웹·지식 그래프에서 더 자주 쓰인다.

택소노미가 "계층"만, 토폴로지가 "연결"만 본다면, 온톨로지는 **연결에 '의미'라는 이름표를 붙인다.** 단순히 "A와 B가 이어졌다"가 아니라, "A는 B를 **소유한다(has-a)**", "A는 B의 **하위 개념이다(is-a)**"처럼 관계의 종류까지 명시한다.

```
개  ──is-a──▶  동물
개  ──has──▶   주인
주인 ──owns──▶ 개
```

이렇게 관계의 종류를 적어 두면, 기계가 **추론**할 수 있다. "개는 동물이다 + 동물은 숨을 쉰다 → 개는 숨을 쉰다"를 사람이 일일이 적지 않아도 끌어낸다. RDF, OWL, 지식 그래프가 이 영역이다.

> 한 줄 요약: **의미 망.** 관계에 이름을 붙여 기계가 추론하게 만든다.

---

## 한 장으로 비교

| | 택소노미 | 토폴로지 | 온톨로지 |
|---|---|---|---|
| 보는 것 | 계층(위아래) | 연결(옆으로) | 의미적 관계 |
| 구조 | 트리 | 그래프(무방향) | 그래프(관계에 이름) |
| 관계 종류 | 하나(상하) | 하나(연결됨) | 여러 개(is-a, has, ...) |
| 대표 분야 | 분류학, 폴더, 카테고리 | 위상수학, 네트워크 | AI, 시맨틱 웹, 지식 그래프 |
| 한 줄 | 줄 세우기 | 연결 지도 | 의미 망 |

세 개를 "관계를 얼마나 풍부하게 담는가" 순으로 늘어놓으면 이렇다.

**택소노미(상하 하나) → 토폴로지(연결 하나) → 온톨로지(관계 여러 개 + 의미)**

같은 대상을 두고도 어디에 초점을 두느냐가 다를 뿐이다. 블로그 글 묶음을 예로 들면 — 카테고리로 나누면 택소노미, 글끼리 링크로 잇는 "관련 글" 그물은 토폴로지, "이 글은 저 글의 **선행 지식이다**"처럼 관계에 의미를 붙이면 온톨로지에 가까워진다.

## 정리

- 헷갈릴 땐 **"이건 줄 세우기(택소노미)인가, 연결 지도(토폴로지)인가, 의미 망(온톨로지)인가?"** 한 가지만 물어보면 된다.
- 셋은 경쟁 관계가 아니라 **초점의 차이**다. 실제 시스템은 보통 셋을 섞어 쓴다.
